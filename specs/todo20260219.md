## Issue Notes and Proposed Solutions

### 1. CORE-ASSETS-DEPENDENCIES-STATUS-TODO (MEDIUM)
Description: `Status::Dependencies` branches still contain `todo!()` in asset execution paths.
Proposed solution:
1. Define explicit transitions for `Dependencies -> Processing/Ready/Error` in `assets.rs`.
2. Implement handler code paths currently behind `todo!()`.
3. Add tests for dependency wait/resume including cancel behavior.

### 2. CORE-INTERPRETER-UNIMPLEMENTED-STEPS (MEDIUM)
Description: `Step::Plan(plan)` and `Step::GetAssetBinary(key)` were unimplemented in `apply_step`, leaving runtime behavior incomplete.
Status: Closed on 2026-02-20.
Implemented:
1. `Step::Plan(plan)` now executes nested plans via `apply_plan(plan, input, context, envref)`.
2. `Step::GetAssetBinary(key)` now resolves the asset through `AssetManager`, loads bytes via `asset.get_binary().await?`, and returns `E::Value::from_bytes(...)`.
Verification:
1. `cargo test -p liquers-core` passed (all tests green).

### 3. CORE-ASSETS-RECIPE-DELEGATION-DEADLOCK (HARD)
Description: potential deadlock when delegating recipe evaluation if queue start/scheduling is not guaranteed.
Proposed solution (high-level): redesign delegation protocol with explicit job ownership and lifecycle signaling; validate with stress tests across cyclic-like dependency graphs.

### 4. CORE-ASSETS-PERSISTENCE-GAPS (HARD)
Description: incomplete write-through semantics for `set_value`, `set_state`, metadata enrichment, and quick-plan paths.
Proposed solution (high-level): define a single persistence contract for asset value+metadata updates, then refactor set/apply code paths to share it.

### 5. CORE-STORE-OPENBIN-MISSING (MEDIUM)
Description: `openbin` unimplemented in multiple store layers.
Proposed solution:
1. Add concrete `openbin` semantics to core store trait docs.
2. Implement in `MemoryStore` and forwarding wrappers.
3. Implement/bridge in `opendal_store`.
4. Add roundtrip tests for binary streaming and metadata preservation.

### 6. STORE-OPENDAL-SLASH-HANDLING-BUG (MEDIUM)
Description: known bug with `'/'` handling in OpenDAL path logic.
Proposed solution:
1. Centralize key<->path normalization and escaping.
2. Replace ad-hoc path manipulation in OpenDAL adapter.
3. Add tests for root key, nested dirs, trailing slash, and listdir parity.

### 7. CORE-PLAN-RELATIVE-RESOLUTION-MISSING (HARD)
Description: relative key/query resolution in dependency discovery is TODO.
Proposed solution (high-level): introduce canonical resolution API for `Key`/`Query` with cwd and use it consistently in planner, dependency scanner, and recipe expansion.

### 8. CORE-PLAN-POLICY-AND-DEFAULTS-TODO (MEDIUM)
