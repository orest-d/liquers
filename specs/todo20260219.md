## Issue Notes and Proposed Solutions

### 1. CORE-ASSETS-DEPENDENCIES-STATUS-TODO
Description: `Status::Dependencies` branches still contain `todo!()` in asset execution paths.
Proposed solution: Implement explicit handling for dependency-wait state transitions; add integration tests covering dependency resolution lifecycle.

### 2. CORE-INTERPRETER-UNIMPLEMENTED-STEPS
Description: `nested plan` and `GetAssetBinary` are unimplemented in interpreter execution.
Proposed solution: Define semantics for nested plan evaluation and binary retrieval; add `Step` contract tests in `interpreter.rs`.

### 3. CORE-ASSETS-RECIPE-DELEGATION-DEADLOCK
Description: FIXME notes possible deadlock when delegating recipe evaluation if queue start is not guaranteed.
Proposed solution: enforce queue scheduling/ownership before delegation and add concurrent recipe-chain deadlock tests.

### 4. CORE-ASSETS-PERSISTENCE-GAPS
Description: TODOs indicate incomplete persistence/update behavior (`set_value`, `set_state`, metadata refresh, quick plans).
Proposed solution: finalize write-through policy and metadata mutation path; add end-to-end tests for set/apply semantics.

### 5. CORE-STORE-OPENBIN-MISSING
Description: `openbin` remains unimplemented in multiple store layers.
Proposed solution: introduce shared binary-stream trait API and implement for MemoryStore and OpenDAL adapters.

### 6. STORE-OPENDAL-SLASH-HANDLING-BUG
Description: Known bug with `/` handling in OpenDAL path logic.
Proposed solution: normalize key/path encoding centrally and add regression tests for root/subdir edge cases.

### 7. CORE-PLAN-RELATIVE-RESOLUTION-MISSING
Description: Relative key/query resolution TODOs in dependency analysis.
Proposed solution: add canonical `resolve_relative(cwd)` methods on key/query and use them in plan dependency traversal.

### 8. CORE-PLAN-POLICY-AND-DEFAULTS-TODO
Description: cache/volatile/inline policy TODOs and questionable default `expand_predecessors=true`.
Proposed solution: define explicit plan builder config defaults, deprecate implicit defaults, and add matrix tests.

### 9. CORE-VALUE-INTERFACE-ARCHITECTURE-DEBT
Description: trait naming/concerns TODOs and serializer configurability TODOs.
Proposed solution: split identifier naming into clearer API (`type_identifier`, `debug_type_name`) and move serializer policy into configurable component.

### 10. CORE-CONTEXT-SESSION-MODEL-TODO
Description: session/user model and context reference TODOs are unresolved.
Proposed solution: add `SessionContext` model (user/session/task ids) and wire it through context creation.

### 11. CORE-COMMAND-METADATA-ENHANCEMENTS
Description: TODOs for global enums/dynamic enums/specialization/input-output typing.
Proposed solution: stage into phases: (1) global enums, (2) dynamic enum provider hook, (3) typed IO metadata.

### 12. CORE-COMMANDS-PERF-CLEANUP
Description: avoidable clone in argument extraction and leftover `PhantomData` uncertainty.
Proposed solution: rework `CommandArguments` to borrow when possible and remove dead type markers after API audit.

### 13. CORE-STATE-LOCK-API-CLEANUP
Description: TODOs indicate internal visibility and locking model need cleanup.
Proposed solution: reduce `pub` surface, replace broad lock exposure with narrower methods and ownership-safe wrappers.

### 14. CORE-METADATA-TRACEBACK-SUPPORT
Description: traceback handling is explicitly unimplemented.
Proposed solution: add optional traceback field with structured error chain format and preserve through serialization.

### 15. CORE-RECIPES-EXPAND-PREDECESSORS-CRASH

Proposed solution: reproduce with focused test, fix plan-builder invariants, and re-enable option safely.

### 16. MACRO-QUERY-VALIDATION-AND-HINTS

### 17. AXUM-ERROR-AND-LISTDIR-SUPPORT-GAPS
